claude code主agent和内置agent是根据提示词模板自动生成的，还是写在代码中实现的
在 Claude Code 中，主代理和内置子代理的实现是一种“代码框架 + 动态提示词注入”的混合模式。
1. 实现机制：代码硬编码 vs. 提示词模板
两者的身份和行为逻辑并不是完全通过单一的“代码写死”或“模板生成”实现的，而是分层协作：
主代理 (Main Agent)：
核心逻辑 (代码实现)：其工具调用循环（Tool-use Loop）、上下文管理、以及如何处理用户输入，是由 Claude Code 的 TypeScript 源码 硬编码实现的。
身份定义 (系统提示词)：其“灵魂”和行为准则（Constitution）存储在内置的系统提示词文件中（约 3100 个 token）。这些提示词在程序启动时被加载并注入到模型请求中。
内置子代理 (Built-in Subagents)：
结构定义 (代码实现)：子代理的启动、任务分配、以及独立上下文窗口的创建是由代码逻辑管理的。
具体职责 (提示词定义)：如 Explore 或 Plan 代理，它们各自拥有独立的 Markdown/YAML 格式的提示词模板。当主代理决定调用它们时，代码会根据模板动态生成该子代理的系统提示词。 

2. 它们是如何“生成”的？
主代理：是常驻的，随 CLI 启动即通过加载内置的 system-prompt 初始化。
内置子代理：是按需生成的。
当主代理识别到任务需要深度检索（Explore）或系统性规划（Plan）时，代码会“生成”一个新的模型实例（子代理）。
这个新实例会加载对应的内置模板作为其身份说明。 
3. 系统提示词的存放位置
在 Claude Code 的分发包中，这些提示词通常以资源文件的形式存在，你可以通过以下方式感知它们的存在：
/agents 命令：此命令会列出所有代理，其背后的描述和逻辑来源于内置的 JSON/YAML 配置文件。
CLAUDE.md 注入：主代理会扫描项目根目录的 CLAUDE.md 文件，并将其中定义的项目规范动态拼接到其系统提示词中，实现基于项目的个性化定制。
自定义代理：如果你使用 /agents 创建新代理，系统会通过一个专门的“代理创建提示词”（Agent Creation Prompt）引导模型生成一段新的 Markdown 提示词，并保存到 ~/.claude/agents/ 目录下。 
总结：主代理和内置子代理的执行流程写在代码里，但它们的身份、能力边界和输出风格是由内置的提示词模板（System Prompts）定义的。

## 核实与补充（基于可验证来源）

上述“代码框架 + 动态提示词注入”的**分层结论是对的**，以下细节根据公开可验证信息做了核对与修正：

### 1. 内置提示词在分发包中的实际形态

- **主代理与内置子代理（Explore / Plan / General-purpose）的提示词**：在官方 npm 包（如 `@anthropic-ai/claude-code`）中，并非以独立 `.md` / `.yaml` 资源文件形式存在，而是**打包在编译后的 JavaScript 中**（约 110+ 条条件加载的字符串，分布在 minified JS 里）。  
- **来源**：社区仓库 [Piebald-AI/claude-code-system-prompts](https://github.com/Piebald-AI/claude-code-system-prompts) 的说明——这些提示词是 “extracted directly from the **compiled minified JavaScript** source code”，即从**编译后的源码**里提取出来的。  
- 因此更准确的说法是：**内置**的主代理与内置子代理的“提示词模板”是**写在代码里、随构建打包进 JS 的字符串**（可能带运行时插值，如 `${GLOB_TOOL_NAME}`），而不是分发包中单独存在的“提示词文件”。  

### 2. “根据模板动态生成”的准确含义

- **内置子代理**：在运行时，代码会**从已打包在程序内的字符串中**选取、拼接或插值，生成该子代理的 system prompt（例如按 agent 类型选不同片段、注入工具名等）。这是“在代码中实现的动态注入”，而不是“从磁盘读取独立模板文件再生成”。  
- **自定义子代理**：用户通过 `/agents` 创建或通过 `~/.claude/agents/` 放置的代理，其提示词才是真正的“独立 Markdown/YAML 文件”，由程序在运行时**从磁盘加载**，这部分符合“提示词模板文件”的直觉。  

### 3. 与上文总结的对应关系

|------|----------|
| 执行流程、工具循环、上下文管理由代码实现 | ✅ 正确 |
| 身份与行为由“系统提示词/模板”定义 | ✅ 正确（内容在逻辑上即“模板”，但内置部分嵌在代码/打包 JS 中） |
| 内置提示词以“资源文件形式”存在于分发包中 | ⚠️ 需修正：内置部分在 **minified JS 内**，不是单独资源文件；仅**自定义**代理使用磁盘上的文件（如 `~/.claude/agents/`） |
| CLAUDE.md 被动态注入、/agents 与自定义代理使用配置与文件 | ✅ 正确 |
