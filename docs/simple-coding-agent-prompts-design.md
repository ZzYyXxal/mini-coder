# 简易 Coding Agent 提示词设计

> **目标**：实现一个具备代码生成、纠错、记忆系统和终端命令执行的简易 coding agent。  
> **参考**：Claude Code CLI 的主代理 + Built-in 子代理（Explore / Plan / General-purpose）架构。

---

## 1. 架构概览

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         主代理 (Main Agent)                              │
│  身份：协调者 + 记忆管理 + 终端执行                                       │
│  能力：理解意图、派发子代理、读写记忆、执行终端（带安全策略）               │
└───────────────────────────────┬─────────────────────────────────────────┘
                                │ 按任务类型派发
        ┌───────────────────────┼───────────────────────┐
        ▼                       ▼                       ▼
┌───────────────┐     ┌───────────────┐     ┌───────────────┐
│   Explorer    │     │    Coder      │     │    Fixer      │
│  （只读探索）  │     │ （代码生成）   │     │ （纠错/调试）  │
│  Read-only    │     │  Write/Edit   │     │  Read+Edit    │
│  快速搜索     │     │  实现需求     │     │  定位并修复   │
└───────────────┘     └───────────────┘     └───────────────┘
```

| 角色 | 身份 | 工具范围 | 何时使用 |
|------|------|----------|----------|
| **主代理** | 协调者、记忆与终端执行 | 全部（含记忆、终端、派发子代理） | 理解请求、派发、汇总、执行命令、更新记忆 |
| **Explorer** | 只读代码库搜索 | Read, Grep, Glob, Bash(只读) | 需要“先搞清楚代码结构/位置”且不修改时 |
| **Coder** | 代码生成与编辑 | Read, Write, Edit, Grep, Glob, Bash(测试等) | 实现新功能、按需求写代码 |
| **Fixer** | 错误定位与修复 | Read, Edit, Write, Grep, Glob, Bash(运行测试) | 根据报错/堆栈定位并修复问题 |

## 2. 设计原则（对齐 Claude Code 结论）

1. **主代理不“单一大段提示词”**：由多段组合（身份 + 工具策略 + 派发规则 + 记忆 + 终端安全）。
2. **子代理独立上下文与专用提示词**：每个子代理只收到自己的 system prompt + 当前任务，不继承主代理完整提示。
3. **派发依据 description**：主代理根据“任务描述”和子代理的 description 决定派给谁；子代理不再派发子代理。
4. **记忆与终端在主代理**：记忆的读写、终端命令的执行与安全策略由主代理统一处理，子代理只产出“可被主代理写入记忆”的摘要（若需要）。
5. **终端安全**：参照项目内 `docs/command-execution-security-design.md`，采用白名单/黑名单 + 用户确认策略，不在提示词中写死命令列表，而是“原则 + 调用权限流程”。

## 3. 提示词文件与用途

| 文件 | 用途 |
|------|------|
| `docs/agent-prompts/main-agent.md` | 主代理系统提示词（组合用） |
| `docs/agent-prompts/subagent-explorer.md` | Explorer 子代理系统提示词 |
| `docs/agent-prompts/subagent-coder.md` | Coder 子代理系统提示词 |
| `docs/agent-prompts/subagent-fixer.md` | Fixer 子代理系统提示词 |

实现时可将上述内容注入到运行时的 system prompt 或对应子代理配置中（如 YAML frontmatter 的 body 或 `prompt` 字段）。

### 使用方式示例

- **主代理**：将 `main-agent.md` 中“身份与角色”至“输出与汇总”各段按需拼接，与工具描述、权限说明等一起组成主代理的 system prompt。
- **子代理**：将各 `subagent-*.md` 的正文（去掉标题与说明）作为该子代理的 system prompt；在配置中为 Explorer 限制为只读工具，Coder/Fixer 开放 Read/Edit/Write 及受限 Bash。

## 4. 记忆系统在提示词中的约定

- **主代理**：会话开始时可根据需要读取记忆（如“最近项目要点、用户偏好”）；子代理返回后，主代理将关键结论写入记忆；提示词中说明“记忆用于跨会话保留关键信息，写入时需简洁、可检索”。
- **子代理**：不直接访问记忆存储；若需要“为记忆提供摘要”，在子代理提示词中要求“在完成时用固定格式输出一段「可写入记忆的摘要」”，由主代理解析后写入。

---

## 5. 终端命令执行在提示词中的约定

- **执行权限**：仅主代理可发起终端执行；子代理 Coder/Fixer 可“请求运行测试或命令”，但由主代理代为执行（或主代理将“运行测试”拆成任务并自己调 Bash）。
- **安全原则**：在主代理提示词中写明：不执行未确认的破坏性命令（如 rm -rf、格式化、批量删除等）；不注入未经验证的外部输入到 shell；长时间或高风险命令需明确提示用户。

具体实现（白名单/黑名单、超时、输出截断）见 `docs/command-execution-security-design.md`，提示词只做“行为约束”与“谁可以调终端”。
